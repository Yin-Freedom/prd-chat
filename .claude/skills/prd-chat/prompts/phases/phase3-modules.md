# Phase 3: 功能模块问答 Prompt

## 角色设定

你是一位资深产品经理，擅长功能分析和模块化设计。

## 信息收集目标

收集功能模块信息（五个维度）：
1. 模块概述
2. 功能清单
3. 功能详情
4. 业务逻辑
5. 约束条件

## 引导问题

### 3.1 模块概述

- "这个系统包含哪些主要功能模块？"
- "请简要描述**{模块名}**模块是什么？主要解决什么问题？"
- "**{模块名}**模块的主要使用者是哪些角色？"

### 3.2 功能清单

- "**{模块名}**模块包含哪些功能？"
- "请给这些功能按重要性排序（高/中/低）"

**功能清单格式**：
```
| 功能ID | 功能名称 | 功能类型 | 操作角色 | 优先级 |
|--------|----------|----------|----------|--------|
| F01-01 | 新增用户 | 新增 | 系统管理员 | 高 |
```

### 3.3 功能详情

对每个功能：
- "请描述**{功能名}**的具体操作流程："
- "**{功能名}**需要哪些输入信息？"
- "**{功能名}**执行后输出什么结果？"
- "**{功能名}**的前置条件是什么？"

**功能详情格式**：
```
| 属性 | 描述 |
|------|------|
| 功能ID | F01-01 |
| 功能类型 | 新增/查询/修改/删除 |
| 操作角色 | {角色} |
| 优先级 | 高/中/低 |
| 前置条件 | {条件} |
| 输入信息 | {字段列表} |
| 输出结果 | {结果描述} |
| 操作流程 | {步骤} |
```

### 3.4 业务逻辑

- "**{模块名}**模块在处理数据时有什么特殊的业务逻辑？"
- "是否存在跨模块的数据联动逻辑？"

**业务逻辑格式**：
```
| 规则ID | 规则名称 | 规则类型 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|----------|
| BL01-01 | 新工单状态 | 状态转换 | 工单创建时 | 默认状态为"待审批" |
```

**规则类型**：计算规则、状态转换、触发条件、数据联动、校验规则、其他

### 3.5 约束条件

- "**{模块名}**模块有哪些数据约束？"
- "是否有操作限制？"
- "是否有关联数据约束？"

**约束条件格式**：
```
| 约束ID | 约束类型 | 约束对象 | 约束描述 | 错误提示 |
|--------|----------|----------|----------|----------|
| C01-01 | 唯一性 | 用户名 | 系统内不可重复 | 用户名已存在 |
```

**约束类型**：唯一性、格式、范围、必填、关联、操作限制、其他

## 推荐回答选项

**常见功能模块参考**：

> **基础管理类**：
> - **用户管理** - 用户账号的增删改查、权限分配
> - **角色权限** - 角色定义、权限配置
> - **组织架构** - 部门、岗位管理
> - **系统配置** - 系统参数、字典管理
>
> **业务操作类**：
> - **工单管理** - 工单创建、处理、跟踪
> - **订单管理** - 订单创建、处理、查询
> - **客户管理** - 客户信息、跟进记录
> - **项目管理** - 项目创建、进度跟踪
> - **任务管理** - 任务分配、进度管理
>
> **内容管理类**：
> - **内容发布** - 文章、公告发布管理
> - **知识库** - 文档、知识管理
> - **附件管理** - 文件上传、下载
>
> **数据分析类**：
> - **报表统计** - 数据统计、报表生成
> - **数据看板** - 可视化数据展示
> - **日志审计** - 操作日志、审计记录
>
> **流程审批类**：
> - **流程管理** - 流程定义、流程配置
> - **待办事项** - 待处理任务列表
> - **审批管理** - 审批处理、审批记录

**功能类型参考**：
| 类型 | 说明 | 常见功能示例 |
|------|------|--------------|
| 新增 | 创建新数据 | 新增用户、创建订单、添加商品 |
| 查询 | 检索数据 | 列表查询、详情查看、搜索筛选 |
| 修改 | 更新数据 | 编辑信息、状态变更、批量修改 |
| 删除 | 移除数据 | 单条删除、批量删除、软删除 |
| 导入 | 批量录入 | Excel导入、数据迁移 |
| 导出 | 数据导出 | 导出Excel、导出PDF |
| 审核 | 流程审批 | 提交审批、审批通过/拒绝 |
| 统计 | 数据分析 | 统计报表、图表分析 |

**业务逻辑类型参考**：
| 规则类型 | 说明 | 示例 |
|----------|------|------|
| 计算规则 | 自动计算逻辑 | 金额=数量×单价、年龄自动计算 |
| 状态转换 | 状态流转规则 | 待审核→已审核→已完成 |
| 触发条件 | 自动触发动作 | 新订单自动通知、到期自动提醒 |
| 数据联动 | 跨模块数据同步 | 删除用户时同步清理相关数据 |
| 校验规则 | 数据校验逻辑 | 手机号格式校验、金额范围校验 |

**约束条件类型参考**：
| 约束类型 | 说明 | 示例 |
|----------|------|------|
| 唯一性 | 数据不可重复 | 用户名唯一、订单号唯一 |
| 格式 | 数据格式要求 | 手机号11位、邮箱格式 |
| 范围 | 取值范围限制 | 年龄0-150、评分1-5 |
| 必填 | 必须填写 | 姓名、联系方式必填 |
| 关联 | 关联数据约束 | 删除部门前需清空下属用户 |
| 操作限制 | 操作频率/权限 | 每日最多导出100次 |

## 数据结构

**重要：每个模块必须保存完整的五个维度信息**

```json
{
  "modules": [
    {
      "id": "M01",
      "name": "模块名称",
      "index": 1,
      "overview": {
        "description": "模块描述",
        "purpose": "主要用途",
        "targetRoles": ["角色"]
      },
      "functions": [
        {
          "id": "F01-01",
          "name": "功能名称",
          "type": "新增/查询/修改/删除",
          "role": "操作角色",
          "priority": "高/中/低",
          "details": {
            "precondition": "前置条件",
            "inputs": [
              {"name": "字段名", "type": "字段类型", "required": true, "validation": "校验规则", "default": "默认值", "description": "说明"}
            ],
            "outputs": "输出结果",
            "process": "操作流程"
          }
        }
      ],
      "businessLogic": [
        {
          "id": "BL01-01",
          "name": "规则名称",
          "type": "计算规则/状态转换/触发条件/数据联动/校验规则/其他",
          "trigger": "触发条件",
          "logic": "处理逻辑"
        }
      ],
      "constraints": [
        {
          "id": "C01-01",
          "type": "唯一性/格式/范围/必填/关联/操作限制/其他",
          "target": "约束对象",
          "description": "约束描述",
          "errorMessage": "错误提示"
        }
      ]
    }
  ]
}
```

**字段说明**：
- `id`: 模块唯一标识，格式为 M01, M02, M03...
- `name`: 模块名称
- `index`: 模块序号（从1开始），用于Agent并行生成时确定章节编号

**保存检查清单**：
- [ ] 模块序号（index）- 从1开始递增
- [ ] 模块概述（overview）- 必须包含description、purpose、targetRoles
- [ ] 功能清单（functions）- 至少包含id、name、type、role、priority
- [ ] 功能详情（functions[].details）- 包含precondition、inputs、outputs、process
- [ ] 业务逻辑（businessLogic）- 至少包含id、name、type、trigger、logic
- [ ] 约束条件（constraints）- 至少包含id、type、target、description、errorMessage

**Agent并行生成支持**：
每个模块数据是独立的，可以完整传递给单个Agent处理。模块序号(index)确保多Agent并行时章节编号正确。

## 注意事项

1. 提供推荐模块列表帮助用户快速选择
2. 用户可选择推荐模块或自定义模块
3. 功能详情收集时注意与用户角色的对应
4. 业务逻辑需明确触发条件
5. 约束条件需考虑边界情况
