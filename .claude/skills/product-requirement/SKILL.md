# 对话式产品需求文档生成 Skill

本skill通过对话式引导问答，帮助用户快速生成产品需求文档和可交互的HTML原型。

## 工作模式

- **引导式问答**：通过结构化问题逐步收集产品信息
- **对话反馈修改**：用户通过对话描述修改需求，重新生成输出
- **迭代式优化**：支持多轮修改直到满意

---

## 数据状态管理

在对话过程中，维护一个结构化的"已收集数据"状态，并**持久化到ClaudeCode记忆工具**中。

### 状态对象模板

```json
{
  "meta": {
    "productName": "",
    "currentPhase": 1,
    "progress": "0/5",
    "moduleProgress": {
      "total": 0,
      "completed": 0,
      "currentModule": null
    }
  },
  "data": {
    "product": null,
    "roles": null,
    "modules": [],
    "flows": null,
    "nonFunctional": null
  }
}
```

### 记忆持久化机制

**状态存储位置**：`{memory-dir}/prd-state.json`

其中 `{memory-dir}` 是 Claude Code 的项目记忆目录，通常位于：
```
~/.claude/projects/{project-path-hash}/memory/
```

**持久化时机**：
1. 每完成一个阶段后立即保存状态
2. **功能模块阶段：每完成一个模块立即保存状态**（细粒度保存）
3. 用户确认修改后更新状态
4. 最终输出生成前保存完整状态

**操作方式**：
- **保存状态**：使用 Write 工具写入 `prd-state.json`
- **读取状态**：使用 Read 工具读取 `prd-state.json`
- **清空状态**：删除文件或重置为初始模板

**状态恢复**：
- 启动skill时，首先尝试读取记忆文件
- 如果存在未完成的状态，询问用户是否继续上次进度
- 如果用户选择重新开始，清空状态文件

### 阶段完成行为

每完成一个阶段后：
1. 更新 `data` 中对应字段
2. 更新 `meta.progress`
3. **使用 Write 工具保存状态到记忆文件**
4. **简要说明已记录**，不重复展示详细对话历史

示例输出：
> ✅ 已记录产品背景并保存进度。
> 💡 **提示**：输入 `/clear` 可清空对话历史，减少上下文占用后继续下一阶段
> 进入阶段2：用户角色...

### 功能模块完成行为（细粒度保存）

在功能模块阶段（阶段3），每完成一个模块后：
1. 将模块数据追加到 `data.modules` 数组
2. 更新 `meta.moduleProgress`（completed +1）
3. **使用 Write 工具立即保存状态到记忆文件**
4. 简要说明已记录该模块

模块确认示例输出：
> ✅ 已记录「用户管理」模块并保存进度（1/3 模块完成）。
> 💡 **提示**：输入 `/clear` 可清空对话历史后继续下一模块
>
> 继续收集下一个模块...
>
> **模块2：订单管理**
> 请简要描述这个模块是什么？主要解决什么问题？

### 修改历史管理

当用户要求修改已收集内容时：
1. 更新对应数据字段
2. 展示修改后的**简要摘要**，而非完整历史

### 进度回顾

当用户输入"回顾"、"进度"、"已完成"等关键词时，展示当前状态：

```
📊 当前进度: 3/5 阶段

✅ 产品背景
✅ 用户角色
🔄 功能模块 (2/3模块完成)
   ✅ 用户管理
   ✅ 订单管理
   ⬜ 报表统计
⬜ 业务流程
⬜ 非功能需求
```

**模块详情查看**：

当用户输入"模块详情"或"已完成模块"时，展示已收集模块的简要列表：

```
📋 已收集模块：

1. **用户管理** (4个功能)
   - 新增用户、查询用户、修改用户、删除用户

2. **订单管理** (5个功能)
   - 创建订单、查询订单、修改订单、取消订单、订单导出
```

### 模块收集流程

```
用户列出模块列表
    ↓
更新 moduleProgress.total = 模块数量
保存状态
    ↓
对每个模块：
    ├── 收集模块概述
    ├── 收集功能清单

    ↓
```

### 操作时机

| 操作 | 触发时机 | 工具 |
|------|----------|------|
| 创建状态 | 启动新会话 | Write |
| 读取状态 | 每次对话开始 | Read |
| 更新状态 | 阶段完成、修改确认 | Write |
| **保存模块** | **功能模块阶段：每完成一个模块** | **Write** |
| 清空状态 | 用户要求重新开始 | Write（初始模板）|

### 状态恢复流程

```
启动skill
    ↓
读取 prd-state.json
    ↓
文件存在？ ──否──→ 创建初始状态，开始新会话
    │
   是
    ↓
状态已完成？ ──是──→ 询问是否开始新需求
    │
   否
    ↓
展示上次进度，询问是否继续
    ↓
用户选择"继续"──→ 恢复状态，继续收集
用户选择"重新开始"──→ 清空状态，开始新会话
```

### 记忆清理

**自动清理时机**：
- 用户明确要求清空进度
- 需求文档生成完成且用户确认满意

**手动清理**：
用户可通过以下方式清空记忆：
- 在对话中输入"清空进度"或"重新开始"
- 直接删除 `prd-state.json` 文件
